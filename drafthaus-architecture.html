Architecture Rules

Rule 1: Never call populateDetail → Use populateSongFields(song)
Rule 2: Dropdowns: position: fixed + getBoundingClientRect()
Rule 3: Sequencer boot: DOM must exist before playback. Use _rowSeqBoot(song). Never call seqPlay() directly unless DOM is mounted.
Rule 4: On auth change: Reset songs, currentId. Stop all polling.
Rule 5: Chord slots: Never use textContent. Always use _seqSlotSetText(slot, x)
Rule 6: After take changes: Always call ftSyncAllStrips()
Rule 7: Playback timing: _ftElapsed() must include _mtPlayStartOffset
Rule 8: Grid logic: Use song.drumSteps and song.chordSteps. Do NOT rely on raw song.grid
Rule 9: Chord resolution fallback: _chordVariantToMidi(name)
Rule 10: CSS layout: Flex + absolute children require height: 100%

Critical Patterns

Sequencer Mapping: Drum and chord grids have different resolutions. Scheduler runs at drum resolution. Chords map via: Math.floor(drumStep * chordSteps / drumSteps)
Four Track Recorder: ft- UI layer shares scope with _mt* and _vr*. RAF loop: ftTickStart() → ftTick()
State Persistence: Firestore only (no localStorage). users/{uid}.lastState, songs/{songId}.uiState
