## 1. Piano Roll re-renders after Sequence Manager bar change

**Problem:** When bars are changed via the Sequence Manager (not the active sequence's own dropdown), `_seqMgrFinishBarsChange` calls `seqBuildGrids` and `prLoadFromGrid` but never calls `prRender()`. The grid data updates but the canvas doesn't repaint ‚Äî user has to refresh the page to see the change.

**Fix:** In `_seqMgrFinishBarsChange` (line ~26396), add `if (typeof prRender === 'function') prRender();` immediately after the existing `prLoadFromGrid` call, inside the `if (idx === seqActiveIdx)` block.

## 4. Chord pill drag to Piano Roll ‚Äî ghost note preview

**Current state:** Chord pills in the left palette column have `draggable="true"` with HTML5 drag data `{chord, degree}`. The existing `dragover` handler on `prNotesLayer` shows a full-height column indicator. The pointer/touch path (`prPalDragStart`) shows a single rectangle spanning the chord's MIDI range.

**Upgrade ‚Äî HTML5 drag path (`dragover` handler):**
- On `dragover`: parse `{chord, degree}` from `e.dataTransfer.getData('text/plain')`. Resolve MIDI notes via `seqGenChordMidi` / `_chordVariantToMidi`. Compute the snapped column from cursor X. Render one ghost div per MIDI note positioned at `(PR_KEY_W + col * PR_COL_W, midiToRow(midi) * PR_ROW_H)`, sized `(prDurToCols() * PR_COL_W) √ó (PR_ROW_H - 1px)`. Ghost style: 50% opacity, purple accent (`#c084fc`), `1px solid #e9d5ff` border, `pointer-events:none`.
- On `dragleave` / `dragend`: remove all ghost divs.
- On `drop`: existing commit logic unchanged ‚Äî notes inserted at snapped column with `prDurToCols()` duration.

**Upgrade ‚Äî pointer/touch path (`prPalDragStart`):**
- Replace the single spanning-rectangle indicator with the same per-note ghost div approach, using the same styling. Ghosts track cursor position on `mousemove`/`touchmove`.

**Ghost container:** A single `div#prChordGhost` appended to `prNotesLayer` (or the grid container), `position:absolute`, `pointer-events:none`, cleared on each move then repopulated. This avoids DOM thrash from individual add/remove per note.

## 7. Bar counter counts continuously across sections

**Problem:** In `_htTickClock`, the line `if (_sectionBars > 0) rawBar = rawBar % _sectionBars;` wraps the bar number back to 0 when a section loops. This causes the UCB bar counter to reset on each section start.

**Fix:** Remove the modulo line entirely. The counter becomes:
```js
var rawBar = Math.floor(totalBeats / timeSig);
// (removed: if (_sectionBars > 0) rawBar = rawBar % _sectionBars;)
const bar  = rawBar + 1 + _barOffset;
```
Bar counter now increments continuously through all sections.
